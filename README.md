Null Gate is a first-person sci-fi portal puzzle game for the browser. It chains together interoperable, fully procedural systems — chambers, physics, apertures, audio — to recreate the feel of a sterile test facility: white panels that accept portals, dark plating that rejects them, weighted cubes on pressure nodes, and momentum flings where falling into a floor portal launches you horizontally out of a wall.

> [!TIP]
> This entire game was generated by an LLM from **a single one-shot prompt**: *"build a Portal-style game with three.js."* Everything after that first pass was refined purely through natural-language feedback — **no human edited the code at any point**.

## Quickstart

```bash
npm install
npm run dev
```

Open the printed URL, click **NEW TEST**, click the scene to capture the cursor — then fire at the bright white panels: **LMB** for blue, **RMB** for amber, and walk through. Press `H` in game for the full control list.

For a production build, use `npm run build && npm run preview`.

> [!TIP]
> Recording a clip? **COMBINED TEST** in the main menu chains every mechanic into one continuous ~60-second route: apertures → test object → pressure node → momentum fling → egress, ending on the **TEST SEQUENCE COMPLETE** screen.

## The Null Gate systems

- **`portal/`**: Portal frames, placement validation and traversal. Position, velocity *and* view direction transform through a single quaternion (`R = out · flip(180°) · in⁻¹`); momentum is preserved (capped at a readable 16 m/s) and falling bodies are gently funneled onto floor portals.
- **`physics/`**: A custom AABB collision world with axis-sweep resolution. Portal traversal works by excluding the one collider a portal sits on, only inside its footprint — the wall is solid everywhere else.
- **`world/`**: The `ChamberBuilder` kit — panel blocks, glass, light strips, hazard stripes, observation rooms, signage — over canvas-generated textures and a shared material library.
- **`player/`**: First-person controller (coyote time, head bob, landing dip) and the procedural aperture-projector viewmodel, whose energy core glows with the last-fired color.
- **`levels/`**: Five hand-tuned chambers — NG-01 First Connection, NG-02 Weighted Trigger, NG-03 Momentum Transfer, NG-04 Final Chamber, and the menu-only NG-X Combined Protocol.
- **`effects/` & `audio/`**: Corkscrewing projectiles with light trails, particle bursts, VSM shadows, Unreal bloom — and every sound synthesized live from WebAudio oscillators.

## Null Gate is fully procedural

There are no asset files in this repository. Wall, floor and stripe textures are drawn on canvases at boot; portal surfaces and borders are GLSL shaders; the viewmodel is composed from three.js primitives; reflections come from a PMREM room environment; all SFX and the facility room tone are oscillator envelopes.

## Why does it read like a real test chamber?

- **Surface language**: white modular panels accept portals, dark plating refuses them — the puzzle is legible from materials alone.
- **Color language**: warm amber means locked or inactive; cool cyan means active or open, consistently across doors, nodes and conduits.
- **Cause and effect**: every pressure node is wired to its door by a glowing floor conduit that ignites when powered.
- **Readable flings**: a high drop, a clearly visible white pit floor, and a distant landing platform let you read the arc before committing; failed attempts respawn you in seconds.
- **Game feel**: tracer flight time, portal opening elastic, screen flash and FOV kick on traversal, tactile button depress, two-leaf doors that slide with weight.
- **Persistence**: unlocked chambers and options (sensitivity, bloom, shadows, raw input) live in `localStorage`.

## Documentation

- **Controls**: `WASD` move · mouse look · `LMB`/`RMB` apertures · `Space` jump · `Shift` sprint · `E` lift/release · `R` restart chamber · `H` controls overlay · `F3` diagnostics · `Esc` pause.
- **Architecture**: each directory under `src/` is one of the systems listed above, plus `game/` (loop, save state, level lifecycle), `ui/` (HUD and menus), `input/` and `utils/`.
- **Reference analysis**: the genre patterns above were extracted from publicly available store pages and gameplay footage of the genre's leading titles; no names, assets, layouts or UI were copied — Null Gate is an original work.

## Known limitations

- Portals render an animated energy field rather than a live view of the other side (a deliberate stability trade-off; the spatial math is fully real).
- Dynamic bodies are non-rotating AABBs — the cube stays upright and cannot be stood on.
- Portals attach to axis-aligned surfaces only, by level construction.
- Audio is intentionally minimal synthesis — functional, not orchestral.
